Error Tracking

Improve debug experience for Reboot app

Error tracking ? Why not logging ?

The logging experience

  1. Log only errors
  2. Read your logs : not enough information
  3. Log everything
  4. Too much noise
  5. Ignore everything

So how error tracking is more beneficial for us ?

Crashes will happen. Most of the time, when you don't expect them.

We need a tool that catches these exeptions for us.

Error names are not sufficients

Catching exceptions means

  • Sending an alert when an unexpected error occurred (failing to read a server data, native crash, problem with input parsing...)
  • Attaching a stack trace
  • Attaching a context (Device type, OS version, App version...)
  • Attaching the history of the user's actions (we will call this "breadcrumbs")

Breadcrumbs

Breadcrumbs can be

  • Service requests (API calls, avatar uploads, assets downloads...)
  • Internal logs
  • Explicit events sent by the App ("Opening SAM", "Adding Friend"...)
  • Previous errors
